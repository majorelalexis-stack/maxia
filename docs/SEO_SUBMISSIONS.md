## Submission workflow

For each directory row above:

1. **Open the submission URL** in browser
2. **Copy standard payload** from the top of this file
3. **Paste into their form**, adjust category if needed
4. **Submit**
5. **Update status** to `submitted` with a date
6. **If approved**: update to `approved` + paste the listing URL
7. **If rejected**: update to `rejected` + note the reason
8. **Follow up** on listings with email confirmation

## Automation opportunities

Some directories have public JSON submission APIs — these can be
automated via a script instead of manual form filling. Candidates for
automation:
- `modelcontextprotocol/registry` — GitHub PR + `server.json`
- `awesome-*` GitHub lists — GitHub PRs via `gh` CLI
- `top.gg` — API submission
- Telegram bot directories — bot API-based

For now everything is manual because most forms require CAPTCHA and
human verification. A bulk-submission script is **not** recommended —
mass submissions are easy to detect and can blacklist the domain.
